03Telegram Mini App
React · Telegram SDK

Основной продукт для клиентов. Запускается внутри Telegram. Авторизация автоматическая — через Telegram аккаунт.

ЭКРАНЫ
  • Онбординг (3 шага) — имя/телефон/цель
  • Главная — баннер, ближайшее занятие, быстрые действия
  • Расписание — фильтр по дням/направлениям, свободные места
  • Запись на занятие — выбор даты → тренер → подтверждение
  • Мои записи — предстоящие + история
  • Тренеры — профиль, расписание тренера
  • Упражнения — материалы от тренеров, фильтры
  • Личный кабинет — профиль, абонемент, настройки
  • Оплата — покупка занятий через Telegram Payments
ФУНКЦИИ
  • Авторизация — автоматическая через Telegram.initDataUnsafe
  • Отмена записи — за 2+ часа до занятия бесплатно
  • Пуш-уведомления — напоминание за 1 час до занятия
  • Отслеживание — счётчик занятий, серии, прогресс
  • Абонементы — покупка и отслеживание баланса занятий
  • Шеринг — пригласить друга через Telegram
  • Offline stub — базовый экран если нет связи
  • Haptic feedback — вибрация на кнопках записи
Флоу регистрации и первой записи
1

Первый запуск Mini App

Telegram передаёт initData с telegram_id, first_name, username. Бэкенд проверяет подпись.

2

Новый пользователь → Онбординг

3 экрана: имя (предзаполнено), телефон (+998 маска), цель (йога/пилатес/всё подряд). Создаётся аккаунт.

3

Главный экран

Баннер "Первое занятие бесплатно" для новых. Ближайшее занятие по выбранному направлению.

4

Запись на занятие

Выбор направления → дата → время → тренер (опционально) → подтверждение. Анимированный успех.

5

Уведомление в Telegram

Бот отправляет сообщение с деталями записи + кнопку "Отменить". Напоминание за 1 час.

Модули панели

  • Контент — упражнения, материалы для Mini App
  • Настройки — студия, пользователи, роли
Аналитика Dashboard
  • KPI метрики — записей сегодня, посещаемость, выручка
  • Заполняемость — % свободных мест по занятиям
  • График посещений — по дням, по направлениям
  • Популярные занятия — топ по записям
  • Удержание — % повторных записей, отток
  • Новые клиенты — по источникам (Landing/Direct/Referral)

  • Отмена занятия с автоматическим уведомлением записавшихся

  • Экспорт расписания в .xlsx / Google Calendar

  • Шаблоны расписания — сохранить и применить на неделю

07Авторизация

Две системы: Telegram для клиентов (Mini App), Email/Password для администраторов (Admin Panel).

Telegram Auth (клиенты)
1

Mini App стартует

Telegram передаёт window.Telegram.WebApp.initData — строка с hash подписью.

2

Фронт → POST /auth/telegram

Отправляет initData. Бэкенд верифицирует HMAC-SHA256 подпись с Bot Token.

3

Бэкенд возвращает JWT

access_token (15 мин) + refresh_token (30 дней). Клиент хранит в memory + localStorage.

4

Все запросы с Bearer токеном

Middleware на каждом маршруте проверяет токен. При 401 — автоматический refresh.

Email Auth (администраторы)
  • bcrypt хэш паролей (cost factor 12)
  • JWT + httpOnly cookie (XSS защита)
  • Rate limiting — 5 попыток в минуту
  • 2FA опционально — TOTP через Authenticator
  • Audit log — каждое действие в admin_audit таблицу
  • Session timeout — 8 часов без активности

08Система уведомлений

Telegram-первичный. SMS — резерв. BullMQ очереди для отложенных задач.

Триггеры уведомлений
  • Запись создана — мгновенно, детали занятия + кнопка отмены
  • Напоминание — за 24 часа и за 1 час до занятия
  • Занятие отменено — мгновенно, причина + альтернативы
  • Место освободилось — из листа ожидания
  • Абонемент истекает — за 3 дня и в день истечения
  • Рассылка от админа — по сегментам клиентов
Архитектура очередей
// BullMQ queues
notifications:immediate  // instant send
notifications:scheduled  // delayed jobs (reminders)
notifications:bulk       // mass sends, rate limited

// Job payload
{
  type: "booking_confirmed",
  user_id: "uuid",
  channel: "telegram",  // | "sms"
  data: { class_id, trainer_name, time }
}
